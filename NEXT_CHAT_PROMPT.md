# 下一轮 Codex 对话提示词

请基于当前文件夹中的 `PRD.md` 和 `TECHNICAL_DESIGN.md`，为一个桌面 AI 笔记管理应用创建工程并实现 MVP。

技术栈要求：

- Tauri
- React
- TypeScript
- Rust
- SQLite
- Markdown 文件作为用户笔记的事实来源

重要约束：

- 先读取 `PRD.md` 和 `TECHNICAL_DESIGN.md`。
- 先输出一份简洁、可执行的实现计划。
- 在我确认计划前，不要开始写代码。
- 第一版不要引入 Python。
- 第一版不要做云同步、多人协作、插件系统、本地大模型或完整 Agent。
- AI 默认只读取当前打开的 Markdown 笔记。
- AI 对文件的写入必须先展示预览或 diff，用户确认后才可以应用。
- Markdown 文件必须继续保存在用户选择的本地文件夹中，不能把正文锁进数据库。
- SQLite 只用于索引、缓存、设置、最近打开记录和元数据。

请按以下顺序实现 MVP：

1. 初始化 Tauri + React + TypeScript + Rust 项目。
2. 配好格式化、lint、typecheck、基础测试命令。
3. 建立主布局：左侧文件树、中间 Markdown 编辑器、右侧 AI 侧边栏。
4. 实现打开本地文件夹作为笔记库。
5. 展示 Markdown 文件树。
6. 打开、编辑、保存 Markdown 文件。
7. 显示保存状态和基础冲突提示。
8. 实现 SQLite 索引和基础搜索。
9. 添加设置页，用于配置 API Key、模型和 AI 读取范围。
10. 实现 AI 侧边栏。
11. 支持总结当前笔记、提取待办、改写选中文本、建议标题和建议标签。
12. AI 写入前展示预览，确认后再写入。

实现时必须遵守 `TECHNICAL_DESIGN.md` 中的约定，特别是：

- 前端组件结构。
- Rust command/API 边界。
- SQLite 表结构。
- 文件读写和外部修改冲突处理。
- AI 请求流式输出方案。
- API Key 安全存储方案。
- 测试策略。

输出要求：

- 先给我一份简洁但具体的实现计划，不要写长篇解释。
- 标明哪些是 MVP 必做，哪些是后续增强。
- 计划里列出将要创建的目录结构和关键命令。
- 等我确认后再开始执行。
